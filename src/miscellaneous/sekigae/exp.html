Technical Breakdown

シャッフルロジック解説

席替えシャッフルツールが内部でどのようにデータを持ち、どのアルゴリズムで席を決定し、どのように演出を実現しているかを解説します。

1

データ構造

座席情報をどのように表現しているか

教室の座席は 行（row）・列（col）・有効フラグ（on） の3つの値を持つオブジェクトの配列として管理されています。 グリッドの最大サイズは ROWS × COLS ですが、on: false にすることで通路や存在しない席を表現できます。

// 座席データの例（6列×7行グリッドの1行分） const seats = [ { r: 0, c: 0, on: true }, // 席あり { r: 0, c: 1, on: true }, { r: 0, c: 2, on: false }, // 通路・空きスペース { r: 0, c: 3, on: true }, ]; // 有効席のみ取り出す（行→列の順にソート済み） const seatList = () => seats.filter(s => s.on).sort((a, b) => a.r - b.r || a.c - b.c);

白 = on:true　点線 = on:false

2

Fisher-Yates シャッフル

偏りのない完全ランダムな並び替えアルゴリズム

出席番号の配列 [1, 2, 3, … n] を Fisher-Yates（Knuth）シャッフル でランダムに並び替えます。 末尾から先頭に向けて走査し、各要素を「それ以前のランダムな位置」と交換するだけのシンプルなアルゴリズムですが、 すべての順列が等確率で出現することが数学的に証明されています。

// Fisher-Yates シャッフル | O(n) で完全ランダム const nums = [1, 2, 3, 4, 5]; // 出席番号配列 for (let i = nums.length - 1; i > 0; i--) { // 0 以上 i 以下のランダムな整数 const j = Math.floor(Math.random() * (i + 1)); // i 番目と j 番目を交換 [nums[i], nums[j]] = [nums[j], nums[i]]; } // 結果例: [3, 1, 5, 2, 4]

3

席への割り当て

シャッフル済みの番号を有効席に前から順に対応させる

Fisher-Yates でランダム化した出席番号の配列を、有効席リスト（行→列順）に先頭から順に対応させます。 結果は assignment オブジェクトで "行_列" → 出席番号 のマップとして保持します。 出席者数が席数より少ない場合は余った席に null が入り空席になります。

// シャッフル済み番号を席に割り当て const shuffled = [3, 1, 5, 2, 4]; const finalAssign = {}; seatList.forEach((seat, i) => { // キーは "行_列" の文字列 const key = `${seat.r}_${seat.c}`; finalAssign[key] = i < useCount ? shuffled[i] : null; }); // 例: { "0_0": 3, "0_1": 1, "0_3": 5, "1_0": 2, "1_1": 4 }
4

ルーレット演出（イーズアウト）

速度を徐々に落として最後に確定するアニメーション

確定した finalAssign はユーザーには見せず、まず全席にランダムな数字を高速で点滅させます。 setTimeout の間隔を 二次関数（イーズアウト） で徐々に延ばすことで、速度が落ちていく演出を実現しています。 合計経過時間が閾値を超えた時点で finalAssign を適用し確定します。

// イーズアウト: 経過時間に応じてtick間隔を伸ばす const totalDuration = 3200; // ms const startInterval = 45; // 最速 (ms) const endInterval = 380; // 最遅 (ms) function getInterval(elapsed) { const p = Math.min(elapsed / totalDuration, 1); // 0 → 1 return Math.round(startInterval + (endInterval - startInterval) * p * p); } function spinTick() { cells.forEach(cell => cell.textContent = Math.ceil(Math.random() * n)); elapsed += getInterval(elapsed); if (elapsed >= totalDuration) { settle(); return; } setTimeout(spinTick, getInterval(elapsed)); }

tick 間隔の変化（二次イーズアウト）

5

紙吹雪（Canvas アニメーション）

requestAnimationFrame による物理シミュレーション

確定と同時に canvas 上に200個のパーティクルを生成します。 各フレームで 重力加速度 を加算し、経過時間が60%を超えると 不透明度をフェードアウト させます。 requestAnimationFrame でディスプレイのリフレッシュレートに同期させることで60fpsの滑らかなアニメーションを実現しています。

// パーティクル初期化 const pieces = Array.from({ length: 200 }, () => ({ x: Math.random() * canvas.width, // X 初期位置（画面幅内ランダム） y: -10 - Math.random() * 300, // Y 初期位置（画面上方 off-screen） vx: (Math.random() - 0.5) * 5, // 水平速度 vy: 1.5 + Math.random() * 4, // 垂直速度（下向き） rot: Math.random() * 360, // 初期回転角 vrot: (Math.random() - 0.5) * 10, // 回転速度 opacity: 1, })); // 毎フレーム更新 function draw(now) { pieces.forEach(p => { p.x += p.vx; p.y += p.vy; p.vy += 0.06; // 重力加速度 p.rot += p.vrot; if (progress > 0.55) p.opacity = Math.max(0, 1 - (progress - 0.55) / 0.45); }); requestAnimationFrame(draw); }

まとめ

データ | 座席を {row, col, on} の配列で管理。on フラグで通路を表現
シャッフル | Fisher-Yates アルゴリズムで O(n) かつ完全ランダム
割り当て | シャッフル済み番号を有効席リストに順番に対応させる
演出 | 二次イーズアウトで加速→減速し確定。Canvas + rAF で紙吹雪
